# ForgeAgent 完整架构文档

本文档为 **ForgeAgent** 全栈应用的核心架构参考，整合目录结构、技术栈、数据模型、Agent 执行流程与 API 设计。实现细节以运行后端 `GET /openapi.json` 为准。

---

## 一、项目概览

**ForgeAgent** 是面向开发者场景的 AI Agent 应用，采用 **Plan-and-Execute** 模式，具备规划、记忆、工具调用与执行四大核心能力。前端提供任务创建与可观测事件监控界面，后端基于 LangGraph 实现 Agent 运行时。

### 技术栈

| 层级             | 技术选型                                                                    |
| ---------------- | --------------------------------------------------------------------------- |
| **前端**         | React 18 + TypeScript + Vite + Tailwind CSS + TanStack Query + React Router |
| **后端**         | Python 3.11+ + FastAPI + SQLAlchemy 2.0 (async)                             |
| **Agent 运行时** | LangGraph（主）+ LangChain（模型/工具/MCP 适配）                            |
| **数据库**       | SQLite（AsyncEngine）+ 独立 LangGraph Checkpoint DB                         |
| **API 风格**     | REST + SSE（Server-Sent Events）                                            |

          │                       │                           │
          ▼                       ▼                           ▼
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────────┐

## 四、Agent 执行流程

### 4.1 Plan-Act-Learn 循环（含 RAG 融合）

```
┌─────────┐
│  START  │
└────┬────┘
     │
     ▼
┌─────────┐    ┌──────────────────────────────────────┐
│ planner │───▶│ SessionLLMContextManager 加载消息    │
└────┬────┘    │ + RAG 知识库自动检索 【新增】       │
     │         │ + 黑板要点拼接                       │
     │         │ + Skill 上下文（如有）              │
     │         │ + plan_steps_with_llm               │
     │         │ + 写入 task_events (plan_created)   │
     │         └──────────────────────────────────────┘
     │
     ▼
┌─────────┐    ┌──────────────────────────────────────┐
│  actor  │───▶│ 逐步执行工具调用：                    │
└────┬────┘    │ - step_start → tool_call → tool_result → step_end │
     │         │ - 可显式调用 rag_search 【新增】     │
     │         │ - 流式总结 llm_stream_delta          │
     │         │ - 支持 max_tool_failure_attempts     │
     │         └──────────────────────────────────────┘
     │
     ▼
┌─────────┐    ┌──────────────────────────────────────┐
│ learner │───▶│ 反思与黑板更新                        │
└────┬────┘    │ - 设置 replan_requested 或失败短路   │
     │         └──────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────┐
│                  条件边路由                       │
├─────────────────────────────────────────────────┤
│ outcome == failed?        → END                  │
│ replan_requested &&      → planner              │
│   未超 max_replan_attempts│ (bump plan_version) │
│ 否则                      → END                  │
└─────────────────────────────────────────────────┘
```

**RAG 融合点**：
1. **Plan 节点**：自动检索 RAG 知识库，注入相关文档到上下文
2. **Actor 节点**：Agent 可显式调用 `rag_search` 工具

| `tool_call`     | 工具调用           | tool_name, arguments              |

| `message`       | 助手消息           | content, is_final                 |

### 7.3 ToolContext 与参数校验

**ToolContext** 通过 `contextvars` 在异步环境中传递用户身份信息：

```python
from app.shared.tool_context import ToolContext, with_tool_context

ctx = ToolContext(user_id="user123", session_id="sess456")
with with_tool_context(ctx):
    # 工具执行时可通过 get_current_tool_context() 获取
    ...
```

**参数校验** 通过 `ToolArgsValidator` 防止工具调用幻觉：

```python
from app.shared.tool_validation import get_tool_args_validator

validator = get_tool_args_validator()
validated = validator.validate("rag_search", {"query": "..."})
```

### 8.2 检索流程

1. **向量检索** → Top-K 向量相似度结果
2. **BM25 检索** → Top-K 关键字匹配结果
3. **RRF 融合** → 倒数排名融合
4. **Rerank** → 交叉编码器重排序
5. **返回最终结果**
