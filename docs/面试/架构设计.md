## 2. 为什么这样分层？

### Q: 为什么 Agent 模块要独立于 API？

| 分层 | 职责 | 好处 |
|------|------|------|
| API | 请求路由、参数校验 | 解耦、可复用 |
| Service | 业务编排、事务边界 | 事务一致 |
| Agent | LLM 调用、状态机 | 核心逻辑独立 |
| Repository | 数据访问 | 数据库解耦 |

### Q: 为什么用异步？

```python
# 同步：阻塞等待
result = await chat.ainvoke(messages)  # 等待 LLM

# 异步：释放线程
# 可并发处理多个请求
```

场景：
- LLM 调用慢（1-3 秒）
- DB 操作可并发
- SSE 轮询长连接

## 3. 数据库设计

### Q: 为什么 TaskEvent 用 seq 字段？

**原因**：SSE 增量推送需要有序序号

```sql
CREATE TABLE task_events (
    id INTEGER PRIMARY KEY,
    task_id TEXT,
    seq INTEGER,  -- 有序序号
    module TEXT,
    kind TEXT,
    payload_json TEXT
);

-- SSE 查询：增量获取
SELECT * FROM task_events
WHERE task_id = ? AND seq > ?
ORDER BY seq
LIMIT 200
```

### Q: 为什么不直接用自增 ID？

| ID | seq |
|----|-----|
| 全局自增 | 任务内自增 |
| 全局唯一 | 按任务独立 |
| 无法支持增量查询 | 支持 `seq > last_seq` |

## 4. 前端架构

### Q: 为什么用 Zustand 而不是 Redux？

| 特性 | Zustand | Redux |
|------|---------|-------|
| 样板代码 | 少 | 多 |
| 学习曲线 | 低 | 高 |
| 性能优化 | 自动 | 需 memo |
| 适用场景 | 中小规模 | 复杂状态 |

ForgeAgent 场景：前端状态不复杂，Zustand 足够。

### Q: SSE 在前端如何处理？

```typescript
// useTaskEvents.ts
const useTaskEvents = (taskId: string) => {
  useEffect(() => {
    const es = new EventSource(`/api/v1/tasks/${taskId}/events/stream`);

    es.addEventListener('plan_created', (e) => {
      store.updatePlan(JSON.parse(e.data));
    });

    es.addEventListener('tool_call', (e) => {
      store.appendToolCall(JSON.parse(e.data));
    });

    return () => es.close();
  }, [taskId]);
};
```

### Q: 熔断器模式实现？

```python
class CircuitBreaker:
    CLOSED = "closed"   # 正常
    OPEN = "open"       # 熔断
    HALF_OPEN = "half_open"  # 半开

    def before_call(self):
        if self._state == OPEN:
            if elapsed > recovery_timeout:
                self._state = HALF_OPEN
            else:
                raise CircuitOpenError()

    def record_success(self):
        self._state = CLOSED
        self._failure_count = 0

    def record_failure(self):
        self._failure_count += 1
        if self._failure_count >= threshold:
            self._state = OPEN
```

## 6. 扩展性设计

### Q: 如何支持多 Agent？

LangGraph 支持子图作为节点：
```python
builder.add_node("sub_agent", sub_graph)

# 主图调用子图
builder.add_edge("planner", "sub_agent")
```

### Q: 如何迁移 PostgreSQL？

只需修改数据库 URL：
```bash
# .env
DATABASE_URL=postgresql+asyncpg://user:pass@localhost/forgeagent
```

SQLAlchemy 2.0 异步驱动自动适配。
