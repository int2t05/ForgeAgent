## 2. 任务状态流转

### Q: 任务有哪些状态？

| 状态 | 说明 | 触发 |
|------|------|------|
| `pending` | 等待执行 | 创建任务时 |
| `running` | 执行中 | 调度到 Agent |
| `success` | 成功完成 | Learner 返回 done 且成功 |
| `failed` | 执行失败 | Actor 异常或超时 |
| `cancelled` | 用户取消 | PATCH /tasks/{id} |

### Q: 状态流转图？

```
创建任务
    │
    ▼
pending ──► running ──► success
                   │
                   ├──► failed
                   │
                   └──► cancelled
```

## 3. SSE 事件流

### Q: SSE 有哪些事件类型？

| 事件 | module | kind | 说明 |
|------|--------|------|------|
| 计划创建 | planning | plan_created | Planner 生成计划 |
| 重规划 | planning | replan | 触发重新规划 |
| 步骤开始 | execution | step_start | Actor 开始执行步骤 |
| 工具调用 | execution | tool_call | 调用工具 |
| 工具结果 | execution | tool_result | 工具返回结果 |
| 步骤结束 | execution | step_end | 步骤执行完成 |
| 反思 | memory | reflection | Learner 生成反思 |
| 任务完成 | workflow | task_complete | 任务结束 |

### Q: SSE 轮询实现？

```python
async def iter_task_event_sse(task_id, after_seq=0):
    last_seq = after_seq

    async with AsyncSessionLocal() as db:
        while True:
            # 1. 查询新事件
            rows = await event_repository.list_events(
                db, task_id, after_seq=last_seq, limit=200
            )

            # 2. 推送事件
            for row in rows:
                yield format_sse(row)
                last_seq = row.seq

            # 3. 检查终态
            task = await task_repository.get_task_by_id(db, task_id)
            if task.status in TERMINAL_STATUSES:
                if no_new_events_for_4_rounds:
                    return

            await asyncio.sleep(0.1)  # 100ms 轮询
```

## 5. 错误处理流程

### Q: 错误处理策略？

| 错误类型 | 处理策略 | 示例 |
|----------|----------|------|
| 瞬时错误 | 指数退避重试 | 429 Rate Limit, 503 |
| 工具超时 | 重试 N 次 | 网络慢 |
| LLM 调用失败 | 重试 + 回退 | API 不可用 |
| 上下文超限 | 截断重试 | Token 超出 |
| 熔断开启 | 快速失败 | 连续失败 |

### Q: 熔断器状态流转？

```
CLOSED (正常)
    │
    ├──► record_failure() × 10 ──► OPEN (熔断)
    │
OPEN (熔断中)
    │
    ├──► 60 秒后 ──► HALF_OPEN (半开)
    │
HALF_OPEN (探测)
    │
    ├──► 成功 ──► CLOSED
    │
    └──► 失败 ──► OPEN
```

## 7. 前端数据流

### Q: 前端如何处理 SSE？

```typescript
// 1. 创建 SSE 连接
const es = new EventSource(`/api/v1/tasks/${taskId}/events/stream`);

// 2. 订阅事件
es.addEventListener('plan_created', (e) => {
  const { steps } = JSON.parse(e.data);
  store.setPlan(steps);
});

es.addEventListener('tool_call', (e) => {
  const tool = JSON.parse(e.data);
  store.appendToolCall(tool);
});

es.addEventListener('step_end', (e) => {
  const { summary } = JSON.parse(e.data);
  store.setSummary(summary);
});

// 3. 任务完成
es.addEventListener('task_complete', (e) => {
  store.setStatus('success');
  es.close();
});
```
