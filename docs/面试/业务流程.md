## 2. 任务状态流转

### Q: 任务有哪些状态？

| 状态 | 说明 | 触发 |
|------|------|------|
| `pending` | 等待执行 | 创建任务时 |
| `running` | 执行中 | 调度到 Agent |
| `success` | 成功完成 | Learner 返回 done 且成功 |
| `failed` | 执行失败 | Actor 异常或超时 |
| `cancelled` | 用户取消 | PATCH /tasks/{id} |

### Q: 状态流转图？

```
创建任务
    │
    ▼
pending ──► running ──► success
                   │
                   ├──► failed
                   │
                   └──► cancelled
```

### Q: 状态转换代码？

```python
# backend/app/services/task_service.py

async def _run_agent_task(task_id: str, session_id: str) -> None:
    try:
        # 更新为运行中
        await task_repository.update_status(db, task_id, TaskStatus.RUNNING)

        # 执行 Agent
        async for update in graph.astream(stream_input, config):
            # 发布事件
            await _publish_task_events(db, task_id, update)

        # 执行完成，检查结果
        final_state = await graph.aget_state(config)
        if final_state.values.get("outcome") == "success":
            await task_repository.update_status(db, task_id, TaskStatus.SUCCESS)
        else:
            await task_repository.update_status(db, task_id, TaskStatus.FAILED)

    except Exception as e:
        logger.exception("Task %s execution failed", task_id)
        await task_repository.update_status(db, task_id, TaskStatus.FAILED, error=str(e))
```

## 3. SSE 事件流

### Q: SSE 轮询实现？

```python
# backend/app/services/event_stream_service.py

async def iter_task_event_sse(task_id, after_seq=0):
    last_seq = after_seq

    async with AsyncSessionLocal() as db:
        while True:
            # 1. 查询新事件
            rows = await event_repository.list_events(
                db, task_id, after_seq=last_seq, limit=200
            )

            # 2. 推送事件
            for row in rows:
                yield format_sse(row)
                last_seq = row.seq

            # 3. 检查终态
            task = await task_repository.get_task_by_id(db, task_id)
            if task.status in TERMINAL_STATUSES:
                if no_new_events_for_4_rounds:
                    return

            await asyncio.sleep(0.1)  # 100ms 轮询
```

### Q: SSE 流式输出优势？

| 指标 | 传统轮询 | SSE 流式 |
|------|----------|----------|
| 首字节延迟 | 等待完整响应 | 立即可见 |
| 用户感知 | 长时间等待 | 实时反馈 |
| 服务器压力 | 高频轮询 | 单连接推送 |
| 用户体验 | 差 | 好（减少 80% 等待感知）|

### Q: 工具参数校验防幻觉？

```python
# backend/app/shared/tool_validation.py

def validate_tool_args(tool_name: str, args: dict[str, Any]) -> ValidationResult:
    """防止工具调用幻觉 (Hallucination)

    问题: LLM 有时会生成不存在的工具或错误参数
    解决: 严格校验工具名和参数类型
    """
    tool_meta = tool_registry._by_name.get(tool_name)
    if not tool_meta:
        return ValidationResult(success=False, error=f"未知工具: {tool_name}")

    # 参数 Schema 校验
    schema = tool_meta.parameters  # JSON Schema
    errors = jsonschema.validate(args, schema)
    if errors:
        return ValidationResult(success=False, error=str(errors))

    return ValidationResult(success=True)
```

## 5. 错误处理流程

### Q: 错误处理策略？

| 错误类型 | 处理策略 | 示例 |
|----------|----------|------|
| 瞬时错误 | 指数退避重试 | 429 Rate Limit, 503 |
| 工具超时 | 重试 N 次 | 网络慢 |
| LLM 调用失败 | 重试 + 回退 | API 不可用 |
| 上下文超限 | 截断重试 | Token 超出 |
| 熔断开启 | 快速失败 | 连续失败 |

### Q: LLM 重试机制？

```python
# backend/app/core/llm_retry.py

async def ainvoke_with_retry(
    chat: BaseChatModel,
    messages: list[BaseMessage],
    settings: Settings,
    max_attempts: int = 3,
) -> BaseMessage:
    """LLM 调用重试 (指数退避)"""

    for attempt in range(max_attempts):
        try:
            return await chat.ainvoke(messages)
        except Exception as e:
            if is_context_limit_error(e):
                # 上下文超限, 不重试
                raise

            if attempt < max_attempts - 1:
                # 指数退避
                delay = (2 ** attempt) * settings.retry_base_delay
                await asyncio.sleep(delay)
                continue
            raise
```

## 7. Checkpoint 持久化流程

### Q: 服务重启后如何恢复任务？

```python
# 1. 启动时检查未完成任务
async def startup_recovery():
    unfinished_tasks = await task_repository.list_unfinished_tasks()

    for task in unfinished_tasks:
        config = {"configurable": {"thread_id": task.id}}

        # 获取 checkpoint 状态
        snap = await graph.aget_state(config)

        if snap and snap.next:
            # 有未完成节点，继续执行
            logger.info(f"恢复任务 {task.id}，从节点 {snap.next} 继续")
            asyncio.create_task(_run_agent_task(task.id, task.session_id))

# 2. 任务中断时保存状态
async def _run_agent_task(task_id, session_id):
    try:
        async for update in graph.astream(stream_input, config):
            await _publish_task_events(db, task_id, update)
    except asyncio.CancelledError:
        # 任务被取消，状态已自动保存在 checkpoint
        logger.info(f"任务 {task_id} 被取消，状态已保存")
        raise
```

### Q: 为什么 Checkpoint 用独立 SQLite？

1. **隔离**：LangGraph 内部结构与业务表分离
2. **性能**：避免与 SQLAlchemy 业务连接竞争
3. **迁移**：未来可切换 PostgreSQL 连接池
