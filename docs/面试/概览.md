# 项目介绍与核心问题

## 1. 项目概述

### Q: ForgeAgent 是什么？

**ForgeAgent** 是面向开发场景的 AI Agent 应用，采用 **Plan-and-Execute** 认知框架，具备规划、记忆、工具、执行四大核心能力。

```
用户输入 → Planner(规划) → Actor(执行) → Learner(反思)
                              ↑              │
                              └──── replan ──┘
```

### Q: 为什么不是通用框架？

区别于 LangChain/LangGraph 等通用框架，ForgeAgent 是针对特定场景优化的应用：
- 内置 Plan-Act-Learn 工作流
- 开箱即用的记忆系统
- 内置工具注册表
- 前端 SSE 监控界面

## 2. 核心问题 10 题

### Q1: Plan-and-Execute vs ReAct 区别？

| 模式 | 特点 | 适用场景 |
|------|------|----------|
| **ReAct** | 思考-动作-观察循环，每步立即执行 | 简单任务、工具调用 |
| **Plan-and-Execute** | 先规划再执行，计划驱动 | 复杂任务、多步骤 |
| **ForgeAgent** | Plan-Act-Learn，加入反思机制 | 开发任务、重规划 |

### Q2: 什么是黑板记忆？

跨任务共享的反思笔记池：
- 每个任务结束后 Learner 生成反思
- 追加到 `blackboard_notes`
- 下个任务的 Planner 读取最后 10 条
- 实现"经验积累"

### Q3: 为什么需要重规划？

当执行失败或 LLM 反思认为需要调整计划时：
```
if can_replan and (actor_replan or llm_request_replan):
    return "planner"  # 回到规划节点
```

### Q4: SSE vs WebSocket 选型？

| 特性 | SSE | WebSocket |
|------|-----|-----------|
| 方向 | 单向服务端推送 | 双向 |
| 实现 | 简单 | 复杂 |
| 重连 | 自动 | 需手动 |
| 适用 | 事件流推送 | 实时交互 |

ForgeAgent 选 SSE：事件流推送场景简单够用。

### Q5: 为什么用 SQLite？

MVP 阶段：
- 零配置、零运维
- 异步 SQLAlchemy 2.0 够用
- 后续可平滑迁移 PostgreSQL

### Q6: Session vs Task 区别？

| 概念 | 说明 |
|------|------|
| **Session** | 会话，一次对话上下文 |
| **Task** | 任务，Session 内的具体任务 |
| **Message** | 消息，Session 内的对话记录 |
| **TaskEvent** | 事件，Task 执行过程中的事件流 |

```
Session ───┬── Task ─── Event
           └── Message
```

### Q7: 什么是 ReAct 循环？

```
while True:
    thought = LLM.think(messages)      # 思考
    if action:
        result = execute_tool(action)   # 执行
        messages.append(observation)     # 观察
    elif final_answer:
        return final_answer             # 结束
```

### Q8: 工具注册表设计？

```python
class ToolRegistry:
    def __init__(self):
        builtins = list_builtin_tools()
        self._tools = builtins

    async def refresh(self, settings):
        # 合并内置 + MCP 工具
        mcp_part = await tools_from_mcp_settings(settings.mcp)
        self._tools = self._merge((builtins, mcp_part))
```

### Q9: 为什么需要熔断器？

防止连续失败时持续调用：
```python
class CircuitBreaker:
    # 失败 10 次后熔断
    # 60 秒后进入半开状态
    # 半开成功恢复，失败继续熔断
```

### Q10: Checkpoint 作用？

LangGraph checkpoint 实现：
- 任务中断恢复
- 状态持久化
- 并发控制
